컴포넌트 맵

MyRadio는 Unity 기본 Toggle, ToggleGroup, Selectable의 연출 구조를 피하고, 상태별로 이름이 붙은 GameObject 필드를 직접 켜고 끄며 라디오 버튼을 표현한다.

문서 구조는 각 컴포넌트를 기준으로 둔다. 앞으로 특정 타입을 수정할 때 해당 타입 섹션을 먼저 열어 보면 현재 의도와 피해야 할 방향을 확인할 수 있어야 한다.

  • 라디오 선택 묶음: MyRadio / MyRadioGroup
  • 상태 표시: MyRadio 직접 제어
  • 참고 타입: MyButton / MySwapper
  • 보류 타입: Selectable

핵심 방향 - 공통 컴포넌트 속성

런타임 UI 헬퍼 컴포넌트는 하나의 GameObject에 같은 헬퍼 타입을 하나만 두는 것을 기준으로 한다. 같은 타입을 여러 개 붙이면 콜백 수집, 상태 캐시, UGUI 래퍼 역할이 서로 겹쳐 Inspector와 런타임 의미가 흐려진다.

DisallowMultipleComponent 적용

컨트롤 본체와 얇은 어댑터 컴포넌트에는 DisallowMultipleComponent를 붙여 중복 부착을 막는다. 필수 의존성이 있는 타입은 기존처럼 RequireComponent를 함께 둔다.

RequireComponent는 같은 GameObject의 필수 의존성 표시에만 쓴다. 구조 생성, 자식 오브젝트 구성, 프리팹 스캐폴딩은 이 속성의 책임이 아니므로 별도 Editor 도구나 템플릿 프리팹으로 다룬다.

상호작용과 상태 - MyRadio / MyRadioGroup 컴포넌트

MyRadio와 MyRadioGroup은 하나의 라디오 선택 묶음을 이룬다. MyRadio는 개별 선택지의 입력과 on/off 상태를 담당하고, MyRadioGroup은 자식 라디오 중 하나만 선택되도록 조율한다.

MyRadio 책임

책임 현재 방향 이유
클릭 영역 MyRadio 자신이 담당 MyButton에 맡기면 on/off 연출 오브젝트와 클릭 오브젝트가 갈라져 프리팹 제작자가 클릭 처리를 중복 배치할 위험이 생긴다.
상태 보관 _state enum on/off 기준 상태와 Normal, Highlighted, Selected, Disabled, 그리고 반대값으로 전환되는 Pressed preview를 하나의 상태값으로 보관한다. IsOn과 IsInteractable은 Inspector에서 조정 가능한 authoring 값이며, 값이 바뀌면 현재 조합에 맞는 상태 GameObject를 표시한다.
상태 출력 명명된 상태 GameObject 필드 연출 표시도 MyRadio가 직접 맡는다. 상태 enum과 표시 오브젝트가 같은 컴포넌트에 있어야 pressed preview처럼 라디오 전용 의미를 Inspector에서 바로 읽을 수 있다.
그룹 통지 GroupInterface.OnClick(this) 상위 그룹의 _radios 배열에 포함된 라디오이면 선택/해제 정책을 그룹에 맡기고, 그룹이 없거나 배열에 포함되지 않으면 단독 토글처럼 현재 on/off를 반전한다.
버튼 공통 기능 IsInteractable, SoundOverrides, HoverInterface MyButton을 직접 붙이거나 그 내부 타입을 참조하지 않고, 버튼 계열 컴포넌트로 기대되는 외부 제어 표면과 사운드/hover 콜백 계약을 MyRadio 안에 복제한다. 단, 라디오에서 switch는 곧 click 동작이므로 외부 ClickSound 선택지로 열지 않고, SoundOverrides.Click은 기본 switch SFX의 대체 사운드로만 쓴다.

입력 처리

  • IPointerEnterHandler: 상호작용 가능하면 hover 상태를 켜고 hover 사운드를 재생한다.
  • IPointerExitHandler: 현재 EventSystem 선택 여부에 따라 Normal 또는 Selected enum 상태로 되돌린다.
  • IPointerDownHandler: 왼쪽 버튼이면 현재 EventSystem 선택 오브젝트로 지정한 뒤 현재 값의 반대값을 미리 보여주는 pressed preview 상태로 바꾼다.
  • IPointerUpHandler: 왼쪽 버튼이면 click 성공 여부가 확정되기 전이므로 현재 논리값 기준의 Normal 또는 Selected enum 상태로 되돌린다.
  • IPointerClickHandler: 왼쪽 버튼이면 OuiClick()을 실행하고, 값이 바뀌면 현재 EventSystem 선택 여부에 따라 반대값의 Selected 또는 Normal 상태로 확정한다.
  • ISelectHandler / IDeselectHandler: 패드/키보드 포커스 진입은 Highlighted가 아니라 Selected enum 상태로 반영하고, 포커스 진입 시에도 hover 사운드를 예약 재생한다. 같은 프레임에 click이 실행되거나 pointer hover 사운드가 이미 재생되면 focus hover 사운드는 취소하며, 포커스 해제는 EventSystem 갱신 타이밍에 기대지 않고 Normal 상태로 내린다.
  • ISubmitHandler: 가짜 pressed 연출 없이 OuiClick()만 실행한다.

Submit은 의미 실행만 한다. Unity 기본 Button은 Submit에서 fade 시간만큼 pressed 전환을 만들지만, Toggle은 의미만 실행한다. 라디오는 선택 상태 변경이 본질이므로 강제 pressed 코루틴을 두지 않는다.

현재 코드 상태

  • 파일: Packages/src/Runtime/MyRadio.cs, Packages/src/Runtime/MyRadioGroup.cs
  • MyRadio 기반 타입: MonoBehaviour
  • MyRadio 필수 컴포넌트: 없음. 단, 클릭 판정을 위해 자식 중 하나 이상은 raycast 가능한 Graphic이어야 한다.
  • MyRadio.State: OffNormal부터 OnDisabled까지 단일 상태 enum으로 보관한다. 내부 로직은 보조 숫자 축이나 변환 함수 없이 현재 _state와 목표 enum 값을 직접 다룬다.
  • _isOn과 _isInteractable은 Inspector에서 초기 선택 여부와 상호작용 가능 여부를 고르는 serialized 값이다. public IsOn과 IsInteractable setter는 상태 표시를 바꾸고, SetState는 enum 상태를 기준으로 두 serialized 값을 다시 동기화한다.
  • MyRadio는 선택지별 대표 이미지와 텍스트를 serialized MyImage[], MyText[] 배열 참조로 받는다. 외부 코드는 Sprite와 Title 설정으로 연결된 이미지·텍스트 배열 전체를 갱신할 수 있고, on/off 연출은 계속 StateObjects가 담당한다.
  • MyRadioGroup의 _radios 배열에 포함된 라디오는 선택 초기화를 그룹이 소유하므로, MyRadio 자신의 InitializerInterface.InitialValue는 적용하지 않는다. 그룹이 없거나 상위 그룹 배열에 포함되지 않은 단독 라디오일 때만 이 초기값을 읽는다.
  • Inspector에 직접 노출되는 Tooltip 문구는 사용자가 바로 읽을 수 있도록 한글을 기본으로 작성한다.
  • MyRadio는 ExecuteAlways로 동작해 에디터에서도 _isOn, _isInteractable 변경과 상태 표시 오브젝트 전환을 즉시 맞춘다.
  • 상속/인터페이스 순서: Unity 제공 그룹, 로컬 제공 그룹 순서로 배치하고 각 그룹 안에서는 이름 ABC 순서
  • 함수 순서: Unity 함수, 상속/인터페이스 함수, 로컬 함수 그룹 순서로 배치한다. 상속/인터페이스 함수 그룹 안에서도 Unity 제공 함수, 로컬 제공 함수 순서로 나누고 각 그룹 안에서는 modifier와 무관하게 함수명 ABC 순서
  • 빌드 확인: dotnet build .\oojjrs.oui.csproj -nologo -v minimal 경고 0개, 오류 0개

MyRadioGroup 책임

  • Inspector의 _radios 배열로 MyRadio 목록을 직접 받는다.
  • SelectionMode 기본값은 Required이며, 항상 하나의 라디오만 켜진 상태를 유지한다.
  • Optional은 하나 이하 선택을 허용해 이미 선택된 라디오를 다시 클릭하면 0개 선택 상태로 만든다.
  • Multiple은 그룹 내 on/off 제한을 풀고, 각 라디오 클릭을 독립 토글처럼 처리한다.
  • 알림이 필요한 선택 변경에서는 개별 라디오를 OuiSetIsOn으로 맞춘다.
  • 초기화나 조용한 동기화에서는 개별 라디오를 OuiSetIsOnWithoutNotify로 맞춘다.
  • CallbackInterface.OnValueChanged(int index, MyRadio radio)로 그룹 선택 변경을 알린다.
  • AllowSwitchOff는 기존 감각을 위한 호환 속성으로 남기며, Required와 Optional 사이를 전환한다.
  • 선택 index는 _index 하나로 관리한다. Required와 Optional에서 Inspector의 _index는 초기 선택값이자 런타임 현재값이며, 외부 InitializerInterface.InitialIndex 값은 Start()에서 한 번만 _index에 적용한다. Multiple에서는 단일 초기 인덱스를 사용하지 않고 첫 번째 on 라디오의 index만 표시한다.

콜백 결정

OuiSetIsOn은 개별 라디오의 notify API이고, IsOn setter는 이 API로 위임한다. OuiSelect는 이미 선택된 index를 다시 요청해도 그룹 콜백을 한 번 호출하고, 실제 on/off 값이 바뀐 개별 MyRadio도 OuiSetIsOn을 통해 콜백을 호출한다. Multiple에서는 콜백 인자로 실제 변경된 라디오의 index와 radio를 넘기며, OuiSetIndexWithoutNotify는 그룹 콜백과 개별 라디오 콜백을 모두 생략하고 시각 상태만 조용히 맞춘다.

구현 메모

  • 라디오 목록은 자식 자동 수집이 아니라 serialized 배열로 결정한다. 배열 순서가 곧 선택 index 순서이며, MySelector의 값 배열과 같은 authoring 감각을 따른다.
  • 같은 GameObject의 callback과 initializer 의존성은 MyTab처럼 Awake()에서 한 번만 캐시한다. OnEnable과 OnValidate는 의존성을 다시 찾지 않고, 현재 _radios 배열을 기준으로 선택 상태만 갱신한다.
  • MyRadio는 클릭과 Start() 초기화 시점에 상위 GroupInterface를 확인한다. 상위 그룹의 _radios 배열에 자신이 포함된 경우에만 그룹 정책에 위임하고, 포함되지 않으면 단독 토글로 동작한다.
  • 그룹은 배열에 든 라디오를 별도로 등록하지 않는다. 따라서 부모 변경 캐시를 유지할 필요가 없고, 현재 구조에서는 OnTransformParentChanged를 추가하지 않는다.
  • MyRadioGroup은 ExecuteAlways로 동작하되, 에디터에서는 OnValidate()로 Inspector 변경을 표시 상태에 반영하고 OnEnable()과 Start()의 초기화 경로는 플레이 모드에서만 실행한다.
  • OnValidate에서는 모드별로 _index를 보정한다. Required는 0 이상의 유효 인덱스로 강제하고, Optional은 0개 선택을 뜻하는 -1을 허용하며, Multiple은 첫 번째 on 라디오의 index로 갱신한다.
  • Editor 코드를 추가하지 않는 범위에서는 _selectionMode와 _index에 한글 Tooltip을 붙여, Optional의 -1이 선택 없음이라는 사실과 Multiple에서 표시되는 index의 의미를 Inspector에서 직접 안내한다.
  • OnEnable에서는 기존 _index가 유효하면 보존하고, 이미 시작된 뒤 다시 활성화된 경우에는 현재 선택을 콜백으로 다시 알린다. OnValidate에서는 Inspector에서 바뀐 _index를 즉시 확인할 수 있도록 현재 선택을 다시 적용하되 외부 초기화 인터페이스는 읽지 않는다.
  • 라디오가 하나도 없으면 DON'T HAVE RADIO. 경고를 출력한다.
규칙과 계산 모델 - MyRadio 상태 표시 필드

MyRadio는 상태별 표시 GameObject를 배열 인덱스가 아니라 StateObjects 묶음 안의 이름 있는 필드로 가진다. 상태 enum과 표시 필드 이름이 1:1로 대응되므로, 프리팹 제작자는 숫자 순서를 외우지 않고 어떤 상태의 표시물인지 Inspector에서 바로 확인할 수 있고, 필요하지 않을 때는 접어서 사운드 설정처럼 숨길 수 있다.

Enum 값 Radio 상태 이름 조건
0 OffNormal 꺼짐, 상호작용 가능, 주목/눌림 없음
1 OffHighlighted 꺼짐, hover 상태
2 OffToOnPressed 논리값은 꺼짐, pointer down 동안 켜짐 전환 preview
3 OffSelected 꺼짐, EventSystem 선택 상태
4 OffDisabled 꺼짐, 상호작용 불가
5 OnNormal 켜짐, 상호작용 가능, 주목/눌림 없음
6 OnHighlighted 켜짐, hover 상태
7 OnToOffPressed 논리값은 켜짐, pointer down 동안 꺼짐 전환 preview
8 OnSelected 켜짐, EventSystem 선택 상태

MySelector 미사용

MySelector는 index 기반 page 표시에는 여전히 유효하지만, MyRadio의 상태는 OffToOnPressed, OnToOffPressed처럼 라디오 전용 의미가 강하다. 배열과 callback을 한 겹 더 두면 제작자가 인덱스 계약을 암기해야 하므로, MyRadio는 상태 표시 GameObject를 직접 가진다.

StateObjects 접힘 그룹

10개 상태 표시 오브젝트를 모두 MyRadio 최상위 serialized 필드로 노출하면 실제 사용 시 Inspector가 너무 길어진다. 그래서 새 프리팹 제작 표면은 StateObjects 구조체 하나로 접을 수 있게 만들고, 테스트 프리팹의 기존 참조값은 프리팹 YAML을 확인해 새 구조로 직접 옮겼다.

배열 래퍼 속성

MyRadio의 대표 이미지와 텍스트는 내부적으로 배열에 전달되지만, 외부 사용자는 단일 선택지 컴포넌트처럼 Sprite, Title 속성으로 값을 넣는 감각을 유지한다. MyImage[]나 MyText[] 반복 호출 자체는 배열 extension으로 줄일 수 있지만, C# extension은 새 속성을 추가할 수 없으므로 속성 표면이 필요한 경우에는 MyRadio가 필요한 배열 버전 setter를 직접 제공한다. 배열 값의 getter는 어떤 슬롯을 대표값으로 삼을지 안정적인 계약을 만들 수 없으므로 제공하지 않는다. 다만 MyImage의 모든 속성과 함수를 2중으로 복제하지 않고, 현재 라디오 사용처에서 의미가 분명한 대표 속성만 공개한다.

현재 UX 리스크는 10개 상태 오브젝트를 여전히 직접 연결해야 한다는 점이다. 다만 Inspector에서는 한 묶음으로 접을 수 있으므로 평상시 화면 점유를 줄인다. 런타임에서는 전환 대상 상태 필드가 null이면 즉시 경고하고, 제작 단계에서는 문서 암기 대신 상태 오브젝트 자동 생성, 이름 검증, null 슬롯 경고를 제공하는 Editor 보조가 필요하다.

상호작용과 상태 - MyButton 컴포넌트

MyButton은 일반 버튼 입력과 사운드 정책의 기존 기준이다. 내부 이미지 참조는 Unity Image가 아니라 MyImage 래퍼를 기준으로 맞춰, 버튼과 라디오가 같은 이미지 갱신 표면을 바라보게 한다. 기본 disabled 동작은 기존처럼 이미지를 숨기며, 버튼별 옵션을 켜면 이미지를 유지한 채 normal/disable 색상으로 전환한다. 텍스트는 계속 normal/disable 색상을 직접 전환한다. 하지만 MyRadio가 MyButton을 직접 조합하지는 않는다.

가져올 수 있는 생각 가져오지 않는 구조
pointer/focus 기반 hover 사운드, click 사운드, per-instance sound override 가능성 Button 기반 클릭 영역과 Unity 기본 Selectable transition
IsInteractable을 외부에서 제어하는 public property 감각 on/off 상태 오브젝트마다 클릭 처리를 붙이는 프리팹 구조
MyButton 조합 미사용

라디오 버튼은 켜짐 상태 자체도 클릭 가능한 표면이어야 한다. MyButton이 클릭 루트를 가져가면 연출 오브젝트와 입력 오브젝트가 갈라져 제작 실수가 늘어난다.

MyRadio는 MyButton을 요구하지 않고, MyButton의 nested type도 직접 쓰지 않는다. 대신 SoundOverrides, HoverInterface, IsInteractable 계약을 MyRadio 소유로 복제한다. 라디오의 click은 on/off 선택 변경 하나만 의미하며 그 동작이 switch이므로, 별도 ClickSound 정책은 만들지 않는다. 기본 재생은 switch SFX이고, 인스턴스별 대체가 필요할 때만 SoundOverrides.Click을 쓴다. 연출 표시는 상태별 GameObject 필드를 MyRadio가 직접 제어한다.

UI와 편의 시스템 - MyTab 컴포넌트

MyTab은 탭 헤더의 단일 선택 상태를 MyRadioGroup에 맡기고, 본문 페이지 전환을 MySelector에 맡기는 얇은 연결 컴포넌트다. 헤더 입력과 선택 표시는 MyRadioGroup 배열에 연결된 MyRadio가 담당하므로 별도 MyTabHeaderButton은 유지하지 않는다.

  • 같은 GameObject의 MyRadioGroup과 MySelector를 RequireComponent로 요구한다.
  • RequireComponent 대상 의존성은 누락 시 복구 가능한 선택지가 아니므로 Awake()에서 한 번만 캐시하고 이후 반복 조회하지 않는다.
  • 탭 헤더는 Required 라디오 그룹으로 설정해 사용하며, MyTab이 런타임에서 모드를 강제로 덮어쓰지 않는다. 잘못 설정된 경우 플레이 모드의 Start()에서만 경고를 출력한다.
  • MyTab은 선택 index를 캐시하지 않는다. 현재 선택값은 필요할 때마다 MyRadioGroup.Index에서 읽고, 본문 MySelector는 그 값에 맞춰 동기화한다. 헤더 라디오 배열과 본문 페이지 배열은 같은 index 순서를 가져야 한다.
  • 외부 코드가 초기 탭을 정해야 할 때는 같은 GameObject의 컴포넌트가 MyTab.InitializerInterface.InitialIndex를 구현한다. MyTab은 MyRadioGroup.InitializerInterface를 내부 구현으로 숨기고, 라디오 그룹이 이 값을 Start()에서 한 번만 읽게 한다.
  • 탭 진입/이탈 콜백은 선택 사항이다. 콜백 컴포넌트가 없는 탭도 정상 구성이므로 Start()에서 별도 경고를 출력하지 않는다.
  • ExecuteAlways로 동작해 에디터에서도 라디오 그룹의 선택 표시와 본문 페이지가 같은 index를 바라보게 한다. 본문 MySelector 동기화는 MyRadioGroup.CallbackInterface.OnValueChanged에서만 시작한다.
UI와 편의 시스템 - MySwapper 컴포넌트

MySwapper는 MySelector와 같은 오브젝트에 붙는 패턴의 선례다. 클릭 입력은 MySwapperButton이 담당하고, MySwapper는 bool 상태를 MySelector index로 바꾸는 얇은 어댑터다.

RequireComponent(typeof(MySelector)) 선례

MySwapper가 RequireComponent를 쓰는 이유는 콜백 인터페이스 연결이 간단해지기 때문이다. MyRadio는 상태 의미가 라디오에 강하게 묶여 있어 이 패턴을 더 이상 따르지 않는다.

MyRadio는 MySwapper처럼 별도 버튼 컴포넌트에 클릭을 맡기지 않는다. 라디오는 on/off 연출 표면 전체가 클릭 가능해야 해서 입력 루트와 상태 계산을 같은 컴포넌트가 갖는 편이 낫다.

미결정과 제외 - Selectable / MySelectable 컴포넌트

Selectable은 패드/키보드 navigation, currentSelectedGameObject, interactable 저장소 측면에서 매력적이었다. 그러나 현재 MyRadio 설계에서는 쓰지 않는다.

Selectable 미사용

Selectable을 상속하거나 요구하면 Transition, Target Graphic, Colors, Sprite State, Animation Triggers처럼 이 설계에서 쓰지 않는 연출 필드가 Inspector에 노출된다. MyRadio의 연출 표시는 명명된 GameObject 필드가 전담하므로 혼란이 크다.

이 결정의 대가는 기본 UGUI navigation 그래프의 자동 이동을 잃을 수 있다는 점이다. SetSelectedGameObject()는 아무 오브젝트나 받을 수 있지만, 방향 이동 탐색은 기본적으로 Selectable 생태계에 기대는 부분이 많다.

MySelectable은 기존 UGUI Selectable의 select/deselect 콜백을 감싸는 타입이다. MyRadio가 Selectable을 쓰지 않는 동안에는 직접 관계를 만들지 않는다.

UI와 편의 시스템 - Editor / Prefab 제작 보조

이 섹션은 런타임 컴포넌트는 아니지만, MyRadio를 실제로 쓰게 만들기 위한 필수 보조 영역이다. 10개 상태 필드를 수동으로 모두 맞추게 하면 UX가 나쁘다.

필요한 기능

  • MyRadio가 요구하는 상태 이름과 개수를 코드 상수로 가진다.
  • Editor 전용 컨텍스트 메뉴나 커스텀 Inspector에서 10개 상태 오브젝트를 생성한다.
  • 생성한 오브젝트를 MyRadio의 명명된 상태 필드에 자동 연결한다.
  • MySelector나 MyRadioGroup에서 자식 오브젝트를 배열로 자동 채우는 기능은 런타임 컴포넌트 본체가 아니라 별도 보조 컴포넌트나 Editor 도구로 제공한다.
  • 필수 상태 필드 중 null 슬롯이 있으면 Inspector에서 즉시 경고한다. 런타임 전환 중에도 현재 켜야 하는 상태 필드가 null이면 DON'T HAVE RADIO STATE {state}. 경고를 출력한다.
  • 상태 오브젝트 이름은 OffNormal, OffHighlighted처럼 인덱스보다 의미가 먼저 보이게 한다.

이 보조 기능이 없으면 MyRadio의 런타임 구조가 맞아도 프리팹 제작 경험은 계속 나쁘다. 다음 구현 단계에서는 MyRadioGroup과 함께 최소 setup/validation UX를 검토한다.

미결정과 제외 - 열린 질문
  • Selectable을 쓰지 않는 상태에서 패드/키보드 방향 이동을 어떻게 보완할지 정해야 한다.
  • MyRadio의 명명된 상태 필드를 Editor에서 자동 구성하려면 context menu로 충분한지, custom inspector까지 갈지 정해야 한다.
  • 현재 MyRadio는 Graphic이 없으면 경고하지만, 클릭용 raycast target을 자동 생성하지는 않는다.
